前言

B站up主霹雳吧啦Wz关于图像分类教程中训练与测试的脚本，因为个人感觉这东西比模型的搭建更重要一些，因此写一篇博客来记录学习的过程，该脚本以2.1 pytorch官方demo(Lenet)为基础并加上3.2 使用pytorch搭建AlexNet并训练花分类数据集训练自己的数据集和6.2 使用pytorch搭建ResNet并基于迁移学习训练迁移学习部分的内容。

1.2 选择训练设备

torch.device选择使用第一块GPU，如果没有则使用CPU。

1.3 数据预处理

训练集的转换：

RandomResizedCrop(224)随机裁剪为224×224大小的图像，并进行缩放；

RandomHorizontalFlip()随机水平反转图像；ToTensor()将PIL图像或NumPy数组转化为张量，并归一化；Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]标准化处理。

测试集的转换：

Resize(256)将图像短边缩放到256像素；

CenterCrop(224)中心裁剪224×224的区域。

1.4 获取数据集

os.path.getcwd返回当前工作目录的路径。

os.path.join将多个路径连接起来，形成一个完整的路径。

os.path.abspath将路径转换为绝对路径。

data_root将当前工作目录与..组合，则返回到上一级工作目录，即存放数据集的工作目录，image_path是数据集的目录。

assert确保数据路径存在，否则抛出错误信息。

dataseys.ImageFolder加载按文件夹组织的图像数据集，每个子文件夹代表一个类别。

train_num输出训练集中的样本数量。

class_to_idx可以获取类别到索引的映射，在花数据集中得到{'daisy':0, 'dandelion':1, 'roses':2,'sunflowers':3,'tulips':4}的字典。

cla_dict = dict((val, key) for key, val in flower_list.items())通过循环遍历将上面字典中键值对反转，如{0:'daisy'}。

json.dumps将字典编码为json格式的字符串，并通过with open写入class_indices.json文件。

1.5 数据加载

这里主要是torch.utils.data.DataLoader的使用，下表是常用的一些参数及作用：

参数 类型 默认值 说明
dataset Dataset 必填 提供要加载的数据集对象，通常是继承自 torch.utils.data.Dataset 的子类
batch_size int 1 每个批次加载的样本数量。
shuffle bool False 是否在每个 epoch 开始前打乱数据。
num_workers int 0 使用多少个子进程加载数据。0 表示数据将在主进程中加载。
drop_last bool False 如果数据集大小不能被 batch_size 整除，是否舍弃最后一个不完整的批次。
collate_fn callable None 用于将一个批次的数据样本合并成一个批次张量的函数。
pin_memory bool False 如果设置为 True，数据加载器会在返回前将张量复制到 CUDA 固定内存中，有助于加速数据传输到 GPU。

1.6 模型加载

resnet34-pre.pth为官方的resnet34预训练权重，可在resnet34-pre.pth下载。

net.load_state_dict用于加载预训练权重，map_location='cpu'代表将权重加载到CPU上。

net.fc = nn.Linear(in_channel, 5)将网络最后一层全连接层的输出修改为5，即花分类数据集的种类个数。

nn.CrossEntropyLoss()代表使用交叉熵损失函数，适用于多分类模型。其他的损失函数可以参考官方文档。

p for p in net.parameters() if p.requires_grad是一个快速创建列表的方法，语法为[表达式 for 变量 in 可迭代对象 if 条件]，即将模型中需要计算梯度的变量p添加到列表中。

optim.Adam表示使用Adam优化器，其他还有SGD、ASGD等优化器，具体用法可以参考官方文档。

1.7 开始训练与验证

设置epoch为30，记录最佳验证准确率best_acc与最佳模型的保存路径save_path，然后循环遍历每一训练轮次，用train_steps记录每一个epoch训练时batch的数量。

在循环训练阶段中，首先用net.train()将模型设置为训练模式，累计损失running_loss初始化为0且后续每一轮次都重置为0，使用tpdm来显示训练进度条。

for step, data in enumerate(train_bar)遍历训练集的每个batch，首先获取图像和对应的标签；使用optimizer.zero_grad()清零优化器梯度，防止与之前batch的计算的梯度相加；再经过前向传播，计算损失，反向传播计算梯度，更新模型参数；最后计算所有batch的累计损失running_loss。

在循环验证阶段中，首先用net.eval()将模型设置为验证阶段，初始准确率acc设置为0并禁止梯度运算；torch.max(output,dim=1)[1]指在output的第一个维度（类别维度）的最大值对应的类别索引。torch.eq逐元素比较两个张量是否相等，然后累加后记录到acc上。

每一epoch结束每个batch的循环训练和验证后，计算平均损失running_loss / train_steps和验证准确率acc / val_num，最后将最佳准确率对应的模型的状态字典保存到save_path中。

2.1 需要import的包

2.2 设备选择与图像处理

这里与训练脚本基本一样，最后通过data_transform和torch.unsqueeze(img,dim=0)将输入的图像转换为张量并拓展第0维度，即增加batch维度，符合模型输入的[1,C,H,W]要求。

2.3 加载类别索引

加载训练脚本中保存的类别索引文件并以只读方式打开，json_load(f)读取json文件内容并解析为一个Python字典。

2.4 创建模型并进行预测

model.eval()切换到预测模式，禁用梯度运算，torch.squeeze()移除张量中维度为1的维度，使输入图像从[1,C,H,W]转换为[C,H,W]。

torch.softmax()对模型输出应用Softmax函数，将输出转化为概率分布。

torch.argmax(predict)返回最大值的索引，即预测类别。

使用plt.title将最大值类别的名字与概率输出到图像标题上，并循环输出每个类别对应的概率，最后打印图片。
